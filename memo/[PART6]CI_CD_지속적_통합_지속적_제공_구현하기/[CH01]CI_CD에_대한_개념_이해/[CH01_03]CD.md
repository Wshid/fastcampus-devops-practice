## [CH01_03] CD

### Continuous Deployment
- 주기적 배포

### Continuous Deployment
- Build server -> Target server
  - 자동 배포가 이루어짐
- 매 빌드마다 자동으로 배포가 일어남
  - 단, 의존성이 생기거나, 마케팅적인 요소가 섞이면 활용 불가

### Continuous Delivery
- Build server -> dockerhub
- human decision -> Target Server

### 관계
- Continuous deployment < continuous delivery

### CI/CD Purpose
- 고객의 니즈를 만족시키 위해 빠르게 배포하기 위함
